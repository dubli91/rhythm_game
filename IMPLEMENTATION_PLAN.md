**Learnings:**

- `timing.ts` control-point semantics: a BPM change at a beat applies before a same-beat STOP (the STOP duration uses the NEW bpm); a note exactly at a STOP's beat sounds when the STOP begins (excludes the STOP duration).
- Global offset is clamped to ±200ms in `audioClock`; formula `songTimeMs = (ctxNow − t0)×1000 − globalOffset + perSongOffset`.
- CN counts as 1 note in `noteCount()`.
- localStorage docs use an envelope `{version, data}` via `createLocalDoc`; keys `settings.v1`/`playOptions.v1`/`records.v1`.

**Learnings / decisions (Milestone 2):**

- Judgement window boundaries are INCLUSIVE with a 1e-6 ms float epsilon (±34ms → GOOD since 34 > 33.33); ±250 BAD, beyond → empty POOR.
- Empty POOR counts toward BP (IIDX convention) but not combo break and not into counts.POOR (missPoor only there).
- HARD gauge <30% mitigation halves ALL decrements, applies when pre-decrement value < 30 (exactly 30 → no halving).
- PlayResult/lamp: clearLampFor(type, cleared, fullCombo); FULL_COMBO overrides; NO_PLAY never returned by it.
- Audio for built-in song is WAV not ogg (no vorbis encoder in env) — spec builtin-song-content MUST 9 deviation, pending; candidate encoders: playwright's bundled ffmpeg (~/.cache/ms-playwright/ffmpeg-1011) or a WASM vorbis encoder in Milestone 4.
- FNV-1a songId hash currently duplicated in scripts/generate-builtin-song.mjs and builtinCharts.test.ts — centralize into src/lib when Milestone 3 needs it for records.
- Playwright is a devDependency used only by scripts/verify-e2e.mjs (not on build path); see .claude/skills/verify/SKILL.md for the headless-Chromium recipe incl. LD_LIBRARY_PATH workaround for missing system libs.

- **beat→ms order of operations** when a STOP and BPM-change coincide at the same beat, or STOP spans a BPM region — algorithm must be pinned (chart-format.md).
- **Global offset sign convention** (added to note time vs input time; +ms = notes feel later/earlier) — must be fixed once and used everywhere (audio-playback.md, judgement-scoring.md).
- **RESOLVED: Empty POOR trigger:** any mapped keydown with no note in that lane's ±250ms window triggers empty POOR; no cooldown. does any keypress with no note in-window penalize, or is there a proximity/cooldown gate — affects gauge fairness (judgement-scoring.md).
- **RESOLVED: Boundary inclusivity** of judgement windows: inclusive with a 1e-6 ms float epsilon (e.g. ±34ms → GOOD since 34 > 33.33). (open vs closed at 16.67/33.33/…); resolve the "±34ms GOOD boundary" wording in the acceptance criteria (judgement-scoring.md).

## Key risks (with mitigations)

- **Mixdown sync fidelity:** one wrong per-note timestamp desyncs the whole track → compute chart-note and mixdown-note times from the exact same `src/lib/chart/timing.ts` function; verify with the autoplay ±10ms acceptance test (bms-import.md).
- **Cross-browser timing precision:** performance.now↔AudioContext.currentTime drift causes judgement error → `getOutputTimestamp()` with calibrated sampling fallback; assert ±2ms median δ stability over 10 replays (audio-playback.md).
- **60fps budget on 2,000-note charts:** per-frame allocation/GC stalls → sprite pooling, culling, no per-frame allocs; measure p95 frame-time ≤16.7ms in 10-min autoplay (playfield-rendering.md).
- **Backgrounded-tab rAF pauses** decoupling notes from audio → derive currentBeat solely from the audio clock every frame; test tab-return alignment (playfield-rendering.md).
- **IndexedDB quota/eviction** losing imported songs → `navigator.storage.persist()`, atomic metadata-last commits, quota-error surfacing, optional compression (song-library.md).
- **Autoplay-policy audio lock** before first gesture → create/resume AudioContext only after first key/click (title screen); handle suspended state (audio-playback.md, app-shell-navigation.md).
- **Large BMS render memory** (long songs, 500+ keysounds) → Web Worker mixdown, evaluate chunked rendering, enforce 50MB/2-min benchmark (bms-import.md).
- **Logic/render entanglement blocking headless tests** → judgement/gauge/scoring pure and event-driven in `src/lib` + features; renderer strictly read-only (judgement-scoring.md, playfield-rendering.md).
