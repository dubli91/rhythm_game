# 곡 라이브러리

> 곡 라이브러리는 내장 곡과 임포트 곡을 로컬에 보관·관리한다.

## 배경

곡 출처는 두 가지다: 앱과 함께 배포되는 **내장 곡**, 사용자가 BMS로 추가하는 **임포트 곡**.
둘은 저장 위치가 다르지만 곡 선택 화면([song-select](song-select.md))에는 동일한 곡 목록으로 합쳐져 보인다.

## 요구사항

### 내장 곡 — MUST

1. 내장 곡은 정적 자산으로 배포한다: `songs/index.json`(곡 목록·메타데이터) + 곡별 채보 JSON + 음원 파일(ogg).
2. 곡 목록 메타데이터는 앱 시작 시 로드하고, 채보·음원은 곡 결정 시(플레이 직전) 지연 로드한다.
3. 최소 3곡 이상의 내장 곡을 포함한다 (난이도 스펙트럼 확인용: 저레벨/중레벨/고레벨).

### 임포트 곡 — MUST

4. [bms-import](bms-import.md)의 결과물(곡 메타 + 채보 + 믹스다운 음원 Blob)을 IndexedDB에 저장한다.
5. 저장 구조는 곡 단위로 삭제 가능해야 한다 (곡 삭제 시 채보·음원 Blob이 함께 제거).
6. 앱 시작 시 임포트 곡 목록을 IndexedDB에서 읽어 내장 곡과 병합한 단일 카탈로그를 제공한다.
7. 임포트 곡 관리 화면: 곡 목록, 개별 삭제, 임포트 진입(드래그&드롭 영역).

### 저장소 안정성 — MUST

8. IndexedDB 쓰기 실패(용량 초과 등)를 감지해 사용자에게 안내한다. 임포트 도중 실패 시 불완전한 곡이 남지 않아야 한다(곡 단위 원자성 — 마지막에 메타데이터를 커밋).

### SHOULD

9. `navigator.storage.persist()` 요청으로 브라우저의 저장소 정리 대상에서 제외되도록 시도한다.
10. `navigator.storage.estimate()`로 사용량/여유 용량을 관리 화면에 표시한다.
11. 곡별 저장 용량 표시 (믹스다운 WAV가 큰 것을 사용자가 인지하도록).

## 데이터 스케치

```
IndexedDB "iidx-web"
├─ objectStore "songs"    : key=songId, value=Song 메타 + charts[]
└─ objectStore "audio"    : key=songId, value=Blob (mixdown WAV)
```

## 수용 기준

- [ ] 임포트한 곡이 새로고침 후에도 목록에 남아 있고 플레이 가능하다.
- [ ] 곡 삭제 후 IndexedDB에 해당 곡의 음원 Blob이 남지 않는다.
- [ ] 임포트 중 탭을 강제 종료해도 다음 실행 시 반쪽짜리 곡이 목록에 나타나지 않는다.
- [ ] 내장 곡은 네트워크 지연이 있어도 곡 선택 목록 표시를 막지 않는다 (메타데이터만 선로드).

## 의존 관계

- 소비자: [song-select](song-select.md) (카탈로그), [audio-playback](audio-playback.md) (음원 로드)
