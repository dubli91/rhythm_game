# 연습곡 콘텐츠 (무음원·단일 키음)

> 연습곡 콘텐츠는 고속 코드+스크래치 복합 패턴을 실전과 동일한 플레이 경로에서 반복 연습하기
> 위한, 음원(BGM) 없이 단일 키음 1종만 사용하는 내장 연습곡 1곡을 정의한다.

**우선순위: 최상위 (2026-07-21 결정)** — 다른 모든 스펙의 미구현 항목보다 본 스펙의 MUST를
먼저 구현한다.

## 배경

2026-07-21 제공된 참조 채보 이미지(원곡 기준 마디 51~66, BPM 282)의 4분음표 코드 연타 +
스크래치 복합 패턴을 연습하려는 요구다. [practice-mode](practice-mode.md) 연습 세션은 최대
8마디·사용자 작성 패턴·게이지/기록 없음에 특화되어 있어 맞지 않는다. 이 패턴은 16마디
분량이며 게이지·판정·기록이 있는 실전 조건 그대로 연습할 수 있어야 하므로 **내장 곡** 형태로
제공한다.

단, 음악(BGM)은 포함하지 않는다. 키 입력 시 재생되는 간단한 키음 샘플 1종만 사용한다 —
[00-overview](00-overview.md) 비목표 "키음 실시간 재생 없음"의 **유일한 예외**이며, 예외
범위는 이 연습곡 1곡·샘플 1종으로 한정한다.

## 요구사항

### 곡 구성 — MUST

1. 연습곡 1곡을 내장 곡 카탈로그(`songs/index.json`)에 추가한다. 곡 선택 화면에 일반 곡과
   동일하게 노출되고, 동일한 플레이 경로(게이지·판정·결과·로컬 기록)를 사용한다.
2. 채보는 난이도 슬롯 1개만 가진다 — [builtin-song-content](builtin-song-content.md) MUST 2
   (슬롯 2개 이상)의 예외이며, MUST 1의 "3곡 이상" 집계에도 포함하지 않는다.
3. BPM 282 고정(변속·STOP 없음), 4/4박자, 총 16마디, tap 노트만 사용한다.

### 패턴 — MUST

4. 패턴은 참조 이미지의 구간 구조를 따르는 3부 구성이다:

   | 구간 | 마디 | 구성 |
   |---|---|---|
   | A 코드 스트림 | 1~9 | 매 박(4분음표) 2~4노트 동시치기 코드. 스크래치는 일부 마디의 첫 박에만 배치(중간에 1·3박 2회 배치 마디 1개 포함) |
   | B 브레이크 | 10~12 | 마디 첫 박 스크래치+코드를 유지하되, 코드 사이에 단타 노트가 섞이는 완급 구간 |
   | C 스크래치 러시 | 13~16 | 거의 매 박 스크래치가 2~3노트 코드와 **동시에** 배치 |

5. 코드 스트림 구간에서 인접한 박의 코드는 레인 구성이 달라져 좌/우 손 위치 이동을 유도해야
   한다 (같은 코드의 단순 반복 금지).
6. 마디 내 정확한 레인 배치는 참조 이미지(2026-07-21 제공, 원곡 마디 51~66)를 옮겨 적어 제작
   단계에서 확정한다. 이미지를 참조할 수 없는 경우에도 MUST 4·5의 구간 구조·코드 규칙을
   만족하면 요구 충족으로 본다.

### 키음·오디오 — MUST

7. 음원(BGM) 파일을 포함하지 않는다. 대신 단일 키음 샘플 1종을 둔다: 짧은 감쇠형 클릭/톤,
   길이 150ms 이하, ogg vorbis 44.1kHz. 자체 제작 또는 CC0 등 퍼미시브 라이선스
   ([builtin-song-content](builtin-song-content.md) MUST 5의 라이선스 규정을 키음에 동일
   적용, `index.json`에 기록).
8. 재생 규칙: 플레이 중 8개 레인(스크래치 포함) 아무 키 입력마다 키음을 즉시 재생한다. 해당
   시각에 노트가 있는지, 판정 결과가 무엇인지와 무관하다(IIDX 키음 방식). 동시 입력 시 중첩
   재생을 허용하고, 볼륨은 음악(music) 게인 계열로 라우팅한다.
9. 마스터 클럭: BGM 소스 노드 없이 [audio-playback](audio-playback.md) MUST 3의 t0 예약
   방식(`t0 = ctx.currentTime + 리드인`)과 MUST 5의 곡 시간 변환을 그대로 사용한다
   (MUST 2 소스 노드 요구의 예외). 곡 종료는 마지막 노트 시각 + 2초 후 정상 종료 경로를 탄다.
10. 키음 버퍼는 곡 결정 시 채보와 함께 프리로드한다. 플레이 중 fetch/decode를 하지 않는다.

### 메타데이터·저장 — MUST

11. `public/songs/<songId>/` 구조를 따르되 `audio.ogg` 대신 키음 파일(예: `keysound.ogg`)을
    둔다. `index.json` 엔트리는 `audio`·`preview` 대신 키음 경로를 기록한다(곡 선택 미리듣기
    없음 — 무음). `songId`/`chartId` 결정적 생성([builtin-song-content](builtin-song-content.md)
    MUST 8)은 동일 적용한다.
12. 채보는 [chart-format](chart-format.md) 스키마 검증·beat→ms 변환을 경고 없이 통과해야
    하며, 내장 채보 전수 유닛 테스트([builtin-song-content](builtin-song-content.md) MUST
    11·12)에 포함한다.

### SHOULD

13. 제목·장르에 연습곡임이 드러나게 한다 (예: 장르 `PRACTICE`).
14. 패턴의 4마디 발췌(A/C 구간 각 1개)를 연습 세션 프리셋([practice-mode](practice-mode.md)
    SHOULD 12)에도 추가해, 구간 단위 루프 연습이 가능하게 한다.

## 수용 기준

- [ ] 곡 선택에서 연습곡을 골라 플레이 → 결과 → 로컬 기록 저장까지 일반 곡과 동일한 경로로
      동작한다.
- [ ] 플레이 시작부터 종료까지 네트워크에 음원 요청이 없고, 키 입력마다(노트·판정 여부 무관)
      키음이 재생된다.
- [ ] 채보가 BPM 282 고정·4/4·16마디이며 스키마 검증·beat→ms 변환을 경고 없이 통과한다.
- [ ] 종반 4마디(C 구간)에 스크래치와 2~3노트 코드가 같은 박에 동시 배치된 박이 매 마디
      존재한다.
- [ ] 키음의 입력→발음 지연이 효과음(메뉴음)과 동급이다 (프리로드된 버퍼 즉시 재생).
- [ ] 같은 곡 10회 반복 플레이에서 판정 δ 평균 편차 ±2ms 이내
      ([audio-playback](audio-playback.md) 수용 기준과 동일 조건, 무음원 모드).

## 의존 관계

- 예외 선언 대상: [00-overview](00-overview.md) 비목표(키음 실시간 재생),
  [builtin-song-content](builtin-song-content.md) MUST 1·2·7·9,
  [audio-playback](audio-playback.md) MUST 2(소스 노드)
- 재사용: [chart-format](chart-format.md), [judgement-scoring](judgement-scoring.md),
  [gauge-clear](gauge-clear.md), [playfield-rendering](playfield-rendering.md),
  [results-records](results-records.md), [song-select](song-select.md)

## 미해결 질문

- 마디별 정확한 코드 레인 배치 — 참조 이미지 기준 제작 단계에서 확정 (MUST 6).
- 표기 레벨 (BPM 282 코드+스크래치 러시 특성상 10~12 범위 예상) — 제작 시 확정.
