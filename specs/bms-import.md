# BMS 임포트 파이프라인

> BMS 임포트 파이프라인은 BMS 패키지를 재생 가능한 내부 포맷 곡으로 변환한다.

## 배경

BMS는 키음 기반 포맷이지만 본 게임은 단일 음원 방식이다. 따라서 임포트 시점에
**채보(노트 타이밍)는 내부 포맷으로 변환**하고, **키음은 OfflineAudioContext로 한 번 렌더링(믹스다운)해
단일 음원**을 만들어 저장한다. 플레이 중에는 단일 트랙만 재생한다.

## 요구사항

### 파싱 — MUST

1. 입력: BMS 폴더의 드래그&드롭(디렉터리) 또는 zip 파일. zip은 브라우저에서 해제한다.
2. 지원 확장자: `.bms`, `.bme`, `.bml`. 폴더 안의 여러 채보 파일은 같은 곡의 난이도들로 묶는다(같은 songId).
3. 텍스트 인코딩: Shift_JIS를 기본으로 시도하고(`TextDecoder('shift_jis')`), UTF-8 폴백. 깨진 문자가 있어도 임포트는 계속한다.
4. 헤더 명령 파싱: `#TITLE #SUBTITLE #ARTIST #GENRE #BPM #PLAYLEVEL #RANK #TOTAL #WAVxx #BPMxx #STOPxx #LNTYPE #LNOBJ`.
5. 채널 파싱:
   - `01` BGM (믹스다운에만 사용)
   - `02` 마디 길이 배율
   - `03` BPM 변경(16진수), `08` 확장 BPM(`#BPMxx` 참조)
   - `09` STOP (`#STOPxx` 참조, 192분음표 단위)
   - 1P 가시 노트 `11~15, 18, 19` → 건반 1~7, `16` → 스크래치
6. `#PLAYLEVEL` → 레벨, `#TOTAL` → 게이지 total. `#TOTAL` 부재 시 기본값 `160 + 총노트수 × 0.16`을 적용한다(추후 조정 가능 상수).
7. 지원하지 않는 명령(`#RANDOM` 계열, BGA 채널 등)은 **무시하되 경고 목록에 수집**하여 임포트 완료 화면에서 보여준다. `#RANDOM`이 포함된 파일은 분기 무시로 채보가 달라질 수 있음을 명시 경고한다.
8. 7키+스크래치로 표현 불가능한 파일(PMS, DP 채널 사용)은 해당 파일만 건너뛰고 이유를 보고한다.

### 믹스다운 — MUST

9. `#WAVxx`가 참조하는 음성 파일을 로드해 디코드한다. WAV/OGG 필수 지원. 확장자가 달라도 실제 파일 탐색 시 `.wav → .ogg` 순으로 대체 탐색한다(BMS 관례).
10. 모든 BGM 노트와 가시 노트의 키음을 각 노트의 절대 시각에 배치해 `OfflineAudioContext`(스테레오, 44.1kHz, 곡 길이 + 테일 2초)로 렌더링한다.
11. 렌더링 결과는 클리핑 방지를 위해 피크 노멀라이즈(피크 ≈ -1dBFS)한다.
12. 결과를 16bit PCM WAV Blob으로 인코딩해 IndexedDB에 저장한다([song-library](song-library.md)).
13. 가시 노트의 시각은 믹스다운에 쓴 절대 시각과 **동일한 계산 결과**를 사용한다 — 채보와 음원의 싱크는 구조적으로 보장되어야 한다.

### UX — MUST

14. 임포트 진행률 표시: 파싱 → 키음 디코드 → 렌더링 → 저장 단계별 진행 상태.
15. 실패 시 어느 파일의 어느 단계에서 실패했는지 사용자에게 보여준다. 부분 성공(여러 채보 중 일부만 변환) 허용.

### SHOULD

16. MP3 키음 디코드 지원 (`decodeAudioData`가 처리 가능한 범위).
17. 저장 용량 절감을 위한 압축 인코딩(WASM Vorbis/Opus 인코더). WAV 저장이 기본이고 압축은 옵션.
18. LN 채널(`51~59` / `#LNOBJ`) → CN 노트 변환.
19. 믹스다운을 Web Worker에서 수행해 UI 프리즈를 방지한다.

## 수용 기준

- [ ] 대표적인 무료 BMS 패키지(폴더에 .bme 3개, ogg 키음)를 드롭하면 하나의 곡 + 난이도 3개로 라이브러리에 나타난다.
- [ ] 변환된 곡을 오토플레이로 재생하면 노트 통과 시점과 해당 키음 소리가 청감상 어긋나지 않는다 (±10ms 이내).
- [ ] BPM 변속·STOP이 있는 BMS에서 음원과 노트가 끝까지 싱크를 유지한다.
- [ ] 키음 파일 일부가 없어도 임포트가 완료되고 누락 목록이 표시된다.
- [ ] zip(50MB, 키음 500개)의 임포트가 일반 노트북에서 2분 이내에 끝난다.

## 의존 관계

- 출력: [chart-format](chart-format.md) 객체 + 믹스다운 음원 → [song-library](song-library.md)에 저장
- UI 진입점: [song-library](song-library.md)의 임포트 화면

- `#STAGEFILE`/자켓 이미지를 곡 선택 화면에 쓸지 (현재는 무시).
