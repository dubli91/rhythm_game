# 플레이필드 렌더링

> 플레이필드 렌더러는 진행 중인 채보를 WebGL로 60fps 스크롤 표시한다.

## 배경

플레이 화면은 PixiJS(WebGL) 캔버스로 그린다. IIDX 1P 스타일 레이아웃(왼쪽 스크래치)을 기준으로 한다.
판정·게이지 로직은 렌더러 밖에 있고, 렌더러는 상태를 읽어 그리기만 한다.

## 요구사항

### 레이아웃 — MUST

1. 기준 해상도 1280×720의 논리 좌표계. 창 크기에 맞춰 비율 유지 스케일링, devicePixelRatio 반영.
2. 레인 구성(왼쪽부터): 스크래치(넓은 레인) + 건반 7개. IIDX 관례 색상 — 스크래치 노트 빨강, 홀수 건반(1·3·5·7) 흰색, 짝수 건반(2·4·6) 파랑.
3. 판정선은 레인 하단에 표시한다. 게이지 바(잔량 + 클리어 라인), 현재 콤보, 판정 텍스트(PGREAT~POOR), EX 스코어, 곡 진행 바, 현재 BPM·하이스피드 값을 HUD로 표시한다.

### 노트 스크롤 — MUST

4. 노트의 화면 y좌표는 **비트 거리 기반**으로 계산한다:
   `y = 판정선Y − (노트beat위치 − 현재beat위치) × 비트당픽셀 × 하이스피드`.
   현재 beat 위치는 곡 시간(오디오 클럭)에서 BPM 이벤트·STOP을 반영해 역산한다. 이 방식으로 스크롤 속도가 자연히 `BPM × 하이스피드`에 비례한다([play-options](play-options.md)).
5. STOP 구간에서는 현재 beat 위치가 멈추므로 노트도 정지한다.
6. 매 프레임 곡 시간은 반드시 오디오 클럭([audio-playback](audio-playback.md))에서 읽는다. `requestAnimationFrame`의 시각을 곡 시간으로 쓰지 않는다.
7. 판정이 끝난 노트는 즉시 제거한다. 미스된 노트는 판정선을 지나 화면 밖으로 흘러나간다.
8. 화면 밖 노트는 그리지 않는다 (가시 구간 컬링).

### 피드백 표시 — MUST

9. 키 입력 중인 레인에 키 빔(세로 발광)을 표시한다 (keydown~keyup 동안).
10. 판정 발생 시 판정 텍스트와 콤보를 판정선 위에 표시한다. PGREAT는 이펙트(플래시)로 구분한다.
11. SUDDEN+ 커버를 레인 위에 오버레이하고 커버 높이 %를 표기한다.

### 성능 — MUST

12. 노트 2,000개 채보에서 일반 노트북 기준 60fps를 유지한다. 노트는 스프라이트 풀링으로 재사용하고 프레임당 할당을 피한다.
13. 프레임 드랍이 발생해도 게임 상태(판정·게이지)는 영향받지 않는다 — 렌더러는 읽기 전용.

### SHOULD

14. CN(차지 노트)의 몸통 렌더링 (시작~끝 구간을 세로로 연결).
15. 폐곡(HARD 0%) 시 화면 정지·페이드 연출.
16. FPS 카운터 토글 (개발·성능 확인용).

## 수용 기준

- [ ] BPM 변속 곡에서 노트 간격이 비트 기준으로 일정하고, 변속 순간 스크롤 속도가 바뀐다.
- [ ] 오토플레이 10분 방치에서 프레임 시간 p95가 16.7ms 이하다 (노트 2,000개 채보).
- [ ] 창 크기를 바꿔도 레이아웃 비율과 판정선 위치가 유지된다.
- [ ] 백그라운드 탭에서 복귀해도 노트 위치가 곡 시간과 일치한다 (rAF 정지의 영향 없음).
